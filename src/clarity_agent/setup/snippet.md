## Clarity Protocol

This project tracks decisions and requirements in `.clarity-protocol/`.

**Before any of these changes, call `check_decision` first:**
new services, auth changes, data schema changes, API contracts, dependency swaps.

**After significant implementation, call `get_packet_status`** to check if docs need updating.

**When the user asks to brainstorm, clarify requirements, or analyze risks,** call `run_clarity`
(or read `{{PROCESSES_DIR}}/clarity-agent.md`).
